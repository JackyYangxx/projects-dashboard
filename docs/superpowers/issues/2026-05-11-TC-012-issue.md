# Issue: TC-012 - Navigation Counter Shows Incorrect Position

**Date:** 2026-05-11T16:40:00+08:00
**Test Case:** TC-012
**Severity:** HIGH
**Status:** RESOLVED

## Description
Navigation counter shows "4 / 3" when viewing the third (last) project in a filtered list of 3 projects. The counter should show "3 / 3".

## Steps to Reproduce
1. Navigate to `http://localhost:5173/`
2. Apply a filter showing exactly 3 projects (status = "进行中")
3. Click on the third project (全球扩张路线图) in the filtered list
4. Observe the navigation counter

## Expected Behavior
- Counter shows "3 / 3" (current position / total)
- Next button is disabled since this is the last project

## Actual Behavior
- Counter shows "4 / 3" (incorrect position)
- Next button is correctly disabled
- The counter is off by 1

## Console Errors
```
No console errors found
```

## Resolution

**Root Cause:** Double +1 conversion - ProjectDetail passed 1-based index but PrevNextNav also added +1 when displaying.

**Fix Applied:**
- Added bounds-checked `displayIndex` in ProjectDetail.tsx
- Removed duplicate `+ 1` from PrevNextNav.tsx counter display

**Commit:** `af023ce`
**Verified:** 2026-05-11
