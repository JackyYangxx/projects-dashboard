# 导入规则修改设计方案

**日期：** 2026-04-21
**状态：** 已批准

---

## 目标

简化 Excel 导入时的校验逻辑，移除不必要的必填字段和状态枚举校验，同时让预算执行率完全由计算得出而非用户输入。

---

## 最终规则定义

### IMPORT_REQUIRED_HEADERS

```ts
// 修改前
['项目名称', '产品线', '负责人', '状态', '项目进展', '总预算', '已用预算']

// 修改后
['项目名称', '产品线', '负责人', '总预算', '已用预算']
```

### 行级处理逻辑

| 字段 | 缺失时行为 |
|------|-----------|
| 项目名称 | 跳过该行 |
| 负责人 | 跳过该行 |
| 产品线 | 空字符串（`''`） |
| 项目进展 | 默认 `0` |
| 总预算 | 默认 `0` |
| 已用预算 | 默认 `0` |
| 状态 | **不导入，忽略 Excel 中的值**，数据库中保持原值 |
| 预算执行率 | **不导入**，由 `已用预算 / 总预算 * 100` 计算得出 |

### 导出逻辑

**不做修改**。导出仍包含 `状态` 和 `预算执行率` 列，供人工查看/编辑。

---

## 改动范围

### 1. `src/constants/project.ts`

- `IMPORT_REQUIRED_HEADERS` 从 7 项缩减为 5 项，移除 `状态` 和 `项目进展`

### 2. `src/pages/Dashboard.tsx`

- `handleImport` 函数中的行级处理逻辑调整：
  - 移除状态列的读取和校验
  - 项目进展缺失时默认 0，不弹窗警告
  - `upsert` 调用时不传入 `status` 字段（保持数据库原有值）

### 3. `src/db/projectDao.ts`（如需调整）

- 确认 `upsert` 函数支持部分字段更新（跳过 status）
